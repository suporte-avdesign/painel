Selects

Plugin necessário

Esta característica requer a arquivos do plugin js/avd.input.js e css/styles/form.css

O formulário do modelo plug-in fornece um estilo de substituição para seleciona, para um estilo uniforme em frente navegadores e plataformas. Para usá-lo, basta adicionar a class: select em qualquer elemento select

<select class="select">
	...
</select>

Seleccione são focusable, e trabalhar com atalhos de teclado também: setas acima e abaixo para navegar na lista de valores, entrar para selecionar, digite o texto para rolar para o primeiro valor correspondente. Em dispositivos de toque, a interface do usuário padrão é usado (a menos que seja desabilitado com uma opção - ver referência abaixo para mais detalhes)

Como a maioria dos elementos de formulário, seleciona pode ser desativado, ou desativando o seleto original ou adicionando a class disabled.

// Disabled
<select class="select" disabled>
	...
</select>

// olhar como disabled
<select class="select disabled">
	...
</select>

Note que se você precisa desativar / select ativado em tempo de execução, você deve usar os métodos de modelo:

// Ativar select
$(selector).enableInput();

// Desativar select
$(selector).disableInput();
Recurso de pesquisa

Desde v1.3, o select pode incorporar um campo de busca que indica sobre o valor selecionado quando o select abre. Por padrão, ele é utilizado automaticamente se o select tem mais de 40 valores, e você pode controlar quando a mostrá-la usando as seguintes opções:searchField, searchIfMoreThan e searchText (ver referência abaixo para mais detalhes).

Opções de classes

Cores

Como qualquer outro elemento, seleciona pode usar os 9 conjuntos de cores:

Abrir em hover

Por padrão, seleciona aberta quando o usuário clica na seta para baixo, mas também podem abrir em hover: basta adicionar a class auto-open.

<select class="select auto-open">
	...
</select>
Lista expansível

No estilo padrão do select, a lista tem a mesma largura que a totalidade de seleção, mas se você quiser a lista para expandir (para textos longos) adicionar a class. expandable-list.

<select class="select expandable-list" style="width:100px">
	...
</select>

Definir uma largura impede o select original a partir de alongamento para corresponder ao valor mais longo.

Check list

Este é um estilo alternativo, ótimo para listas curtas, e é mais amigável ao toque:

<select class="select check-list">
	...
</select>

Multiple selects

Estilo padrão

Multiple selects parece com isso:

<select class="select" multiple>
	...
</select>

Check-list style:

<select class="select check-list" multiple>
	...
</select>

Pode controlar o número de elementos visíveis, utilizando o tamanho de atributo..

Fácil multiple selection

Várias seleciona trabalhar como várias seleciona padrão, segurando Ctrl ou Cmd para selecionar vários valores. Para tornar mais fácil para os usuários a selecionar vários valores, adicione a class easy-multiple-selection, e a seleção só pode ser feita clicando (note que este é o comportamento padrão em dispositivos de toque)

<select class="select easy-multiple-selection check-list" multiple>
	...
</select>

Por padrão, a class easy-multiple-selection não permite selecionar todos os valores, mas isso pode ser habilitado usando a class allow-empty

<select class="select easy-multiple-selection allow-empty check-list" multiple>
	...
</select>
Força styles

É possível exibir uma seleção múltipla como um drop-down usando a class multiple-as-single, aqui com alguma das opções acima:

<select class="select multiple-as-single easy-multiple-selection check-list" multiple>
	...
</select>

No oposto, single-value selects podem ser exibidos como múltiplos usando a class multiple (mas ainda permitir que apenas um value)

<select class="select multiple check-list">
	...
</select>

Opções de referência

Os padrões podem ser modificados em tempo de execução em $.fn.styleSelect.defaults, e as opções podem ser passadas em linha usando data-select-options atributo

styledList
false usar drop-down UI do sistema, fiel ao estilo de usar menus drop-down
Type: boolean|null
Default: true
styledOnTouch
Para dispositivos de toque: false usar drop-down UI do sistema, true para usar estilos drop-downs ou nulos de adivinhar (true para o estilo check-list, false para os outros)
Nota: só funciona se styledList for true
Type: boolean|null
Default: null
openOnKeyDown
Quando focused, deve a seta para baixo abrir o drop-down ou apenas rolar valores?
Type: boolean
Default: true
noValueText
Texto para multiple select e sem valor selected
Type: string
Default: ''
singleValueText
Texto para multiple select com um valor selecionado, ou false para apenas exibir o valor selecionado
Type: string|boolean
Default: false
multipleValuesText
Texto para multiple select com vários valores selecionados, ou false para exibir apenas a lista selecionada
Dica: usado %d como placeholder para o número de valores
Type: string|boolean
Default: '%d selected'
allValuesText
Texto para múltipla escolha com todos os valores selecionados, ou false para exibir apenas a lista selecionada
Dica: usado %d como placeholder para o número de valores
Type: string|boolean
Default: 'All'
searchField
Ativar campo de pesquisa quando aberto - usar null para usar automaticamente quando a lista tem searchIfMoreThan elements
Type: boolean|null
Default: null
searchIfMoreThan
Número mínimo de elementos para provocar um campo de busca, se searchField é null
Type: int
Default: 40
searchText
Texto auxiliar para campo de pesquisa
Type: string
Default: 'Search'
